# PRD: 회의록 → 실행 항목 변환기

## 1. 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 회의록 → 실행 항목 변환기 (가칭: MeetFlow) |
| 목표 | 회의 후 정리의 고통을 제거하고, 실행 가능한 항목으로 자동 변환 |
| 타겟 사용자 | 소규모 팀 (2~10명) |
| MVP 목표 시점 | 1주일 내 (2026.03.27) |
| 핵심 가치 | 회의 끝 → 5초 만에 정리 완료 |

## 2. 문제 정의

- 회의가 끝나면 아무도 정리하고 싶어하지 않는다
- 정리하더라도 누가, 무엇을, 언제까지 해야 하는지 빠지기 쉽다
- 정리된 내용이 팀에 공유되지 않아 후속 조치가 누락된다
- 회의가 반복될수록 미결 사항이 묻힌다

## 3. 핵심 기능 (MVP)

### 3.1 입력

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 텍스트 붙여넣기 | 회의록을 직접 붙여넣어 분석 | P0 (현재 완료) |
| 파일 업로드 | .txt, .docx, .pdf 파일 업로드 | P0 |
| 음성 파일 업로드 | .mp3, .wav 등 업로드 → STT 변환 후 분석 | P1 |

### 3.2 AI 분석 (Claude API)

회의록 입력 시 자동으로 6가지 항목을 추출:

| 항목 | 설명 |
|------|------|
| 결정 사항 | 확정된 내용 + 중요도 (high/medium/low) |
| 미결정 사항 | 아직 결론 나지 않은 건 + 미결 사유 |
| 담당자별 할 일 | 누가 / 무엇을 / 언제까지 |
| 일정 리스크 | 지연·위험 요소 + 심각도 |
| 후속 질문 | 추가 확인이 필요한 사항 |
| 공지/메일 초안 | 바로 발송 가능한 수준의 요약문 |

### 3.3 결과 활용

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 화면 확인 + 전체 복사 | 결과를 보고 클립보드에 복사 | P0 (현재 완료) |
| 링크 공유 | 분석 결과 고유 URL 생성, 팀원에게 공유 | P0 |
| Slack 전송 | Slack Webhook으로 채널에 요약 전송 | P1 |
| 이메일 발송 | 공지 초안을 이메일로 바로 전송 | P1 |
| 히스토리 저장/검색 | 과거 회의록 저장 + 키워드 검색 | P1 |
| 팔로업 이메일 알림 | 할 일 기한 전 담당자에게 리마인더 | P2 |

### 3.4 회의 유형

다양한 회의를 모두 지원:
- 주간 정기 회의
- 프로젝트 회의
- 브레인스토밍/아이디어 회의
- 의사결정 회의
- 1:1 미팅

## 4. MVP 범위 (1주일)

### Phase 1 — 즉시 (Day 1~2): 핵심 입력 강화
- [x] 텍스트 붙여넣기 + AI 분석 + 결과 화면
- [ ] 파일 업로드 (.txt, .docx, .pdf)
- [ ] 결과 링크 공유 (고유 URL)

### Phase 2 — 중반 (Day 3~4): 저장 + 공유
- [ ] 히스토리 저장/검색 (브라우저 로컬 또는 경량 DB)
- [ ] Slack Webhook 전송
- [ ] 이메일 발송 (공지 초안)

### Phase 3 — 마무리 (Day 5~7): 음성 + 알림
- [ ] 음성 파일 업로드 + STT 변환
- [ ] 팔로업 이메일 알림
- [ ] UI 다듬기 + 모바일 반응형

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 인증 | MVP에서는 로그인 없이 사용 (2차에서 Google OAuth 추가) |
| 성능 | 분석 응답 10초 이내 |
| 보안 | API Key 서버사이드 처리, 회의 데이터 암호화 저장 |
| 접근성 | 모바일 반응형 지원 |
| 배포 | Vercel 배포 |

## 6. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js 16 + TypeScript + Tailwind CSS 4 |
| 백엔드 | Next.js API Routes |
| AI | Claude API (Anthropic SDK) |
| STT | Whisper API 또는 Google Speech-to-Text |
| DB | Supabase (히스토리 저장, 링크 공유) |
| 이메일 | Resend 또는 SendGrid |
| 배포 | Vercel |

## 7. 사용자 플로우

```
[회의 종료]
    ↓
[회의록 입력] ← 텍스트 / 파일 / 음성
    ↓
[AI 분석] (5~10초)
    ↓
[결과 화면] → 결정사항 / 미결정 / 할 일 / 리스크 / 질문 / 공지초안
    ↓
[활용]
  ├→ 전체 복사
  ├→ 링크 공유
  ├→ Slack 전송
  ├→ 이메일 발송
  └→ 히스토리 저장
```

## 8. 성공 지표

| 지표 | 목표 |
|------|------|
| 분석 완료율 | 입력 대비 90% 이상 성공 |
| 분석 품질 | 핵심 결정사항·할 일 누락 10% 미만 |
| 재사용률 | 같은 팀에서 주 2회 이상 사용 |
| 공유율 | 분석 결과의 50% 이상이 팀에 공유됨 |

## 9. 향후 로드맵 (2차 버전)

- Google/GitHub OAuth 로그인
- 팀 워크스페이스 (팀별 회의록 관리)
- 실시간 회의 연동 (Zoom, Teams, Google Meet)
- 브라우저 실시간 녹음 + STT
- Notion, Jira/Linear 연동
- 자동 팔로업 알림 고도화 (Slack DM + 이메일)
- 회의 간 연결 (이전 미결 사항 자동 추적)
